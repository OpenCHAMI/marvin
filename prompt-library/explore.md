Explore Phase Contract

Goal
- Build an accurate understanding of the task and codebase before planning.

Scope
- Read-only exploration of repositories, configs, docs, and artifacts.
- Identify reusable patterns, constraints, and unknowns.

Hard Boundaries
- Do not edit source files.
- Do not run source-mutating commands.
- Do not assume architecture from issue text without code evidence.

Required Outputs
- Key findings grounded in file evidence.
- Risks, ambiguities, and dependency hotspots.
- A short list of clarifying questions only when needed.

Quality Bar
- Prefer concrete references over general observations.
- Distinguish confirmed facts from assumptions.
- Keep findings concise and implementation-relevant.
